# pukiwiki自作プラグイン ogp.inc.php
はてなブログの「ブログカード」は魅力的でPukiwikiにも導入したいと思い、自分で見よう見まねでプラグインを作成して個人用のPukiwikiで利用していましたが、せっかくですのでこの外部のOGPを取得してブログカードとして表示するPukiwikiプラグインを公開することにしました。

詳しくはこちらのサイトもご覧ください。
https://oncologynote.jp/?12173bf026

- Pukiwiki 1.5.3および1.5.4RC、PHP 7.4およびPHP 8.0で動作確認しています。
- プラグイン本体はGPL v3です。

## 設置方法

1. まず、[opengraph.php](https://github.com/scottmac/opengraph/) を入手してpluginフォルダに設置します。
2. ogp.inc.phpをpluginフォルダに設置します。
3. cacheフォルダにogpのフォルダを作成し、書込み可能なパーミッションを与えます（666など）。
4. スタイルシートは適宜調整して見栄えを良くします。参考までに、スタイルシートの一例も添付しています。

## 使い方

>\#ogp(取得したいURL)

第2引数以降にnoimgを入れることで画像を非表示にできます。（ver1.5以降）

※ 過去のバージョン履歴で`prefetch`や`amp`といった追加キーワードに触れていますが、現行の公開コードでは`noimg`以外は実装されていません。
  これらのキーワードを指定しても動作に影響はありません。

### WebP fallback

`ogp.inc.php` の先頭で `PLUGIN_OGP_WEBP_FALLBACK` を `TRUE`（既定値）にしておくと、OGP画像をキャッシュする際に JPEG/PNG/GIF などの元フォーマットと WebP 版の2種類を同時に生成します。ブラウザが WebP に対応していない場合でも `<img>` には従来形式を必ず指定するため、WebP 非対応環境でレイアウトが崩れる心配はありません。WebP に対応したブラウザでは `<picture>` の `<source type="image/webp">` が優先されるため転送量を抑えられます。

過去バージョンで作られた JPEG/PNG/GIF のキャッシュしか存在しない場合でも、プラグインが自動的に WebP 版を追加生成してくれるため、設定を切り替えた直後から WebP fallback の恩恵を受けられます（サーバー側で GD の WebP サポートが有効である必要があります）。

### HTTP リクエストのカスタマイズ

`opengraph.php` はブラウザに近い User Agent と Accept 系ヘッダーを付与した cURL でページを取得し、リダイレクト追従・gzip 展開・HTTPS 検証を行うようになりました。OGP 取得時に特定の User Agent を指定する必要がある場合は、`ogp.inc.php` 冒頭の `PLUGIN_OGP_HTTP_USER_AGENT` に任意の文字列を設定してください（空文字なら既定の UA を利用します）。

### 取得失敗時の動作（ver3.0以降）

サーバーからのOGP取得は、bot検知の強いサイト（学術誌サイトなど）やJavaScriptレンダリングが必須のサイトで失敗することがあります。ver3.0からは取得に失敗しても**リンクは消えず**、URLをそのまま表示するプレーンなリンクカード（`.ogp-fallback` クラス）にフォールバックします。失敗したURLはエラーキャッシュとして記録され、30日間は再取得を試みません（サーバー負荷と表示遅延の防止）。

確実に取得できないと分かっているドメインは `PLUGIN_OGP_SKIP_DOMAINS` にホスト名を追加すると、最初から取得を試みずフォールバック表示になります。

### repairモード（ver3.0以降）— ブックマークレットでキャッシュを修復する

取得に失敗したURLのキャッシュを、**あなた自身のブラウザを取得エンジンにして**修復できる管理機能です。ブラウザで普通にページを開いてメタタグを読み取るため、bot検知もJSレンダリングも自然に通過します。追加のソフトウェアは不要です。

**前提**: `pukiwiki.ini.php` の `$adminpass` が設定されていること（未設定のサイトでは無効）。凍結専用サイト等で使わない場合は `PLUGIN_OGP_REPAIR` を `FALSE` にすると機能ごと無効化できます。

**使い方**:

1. `https://あなたのサイト/?plugin=ogp` を開き、管理者パスワードでログインする
2. 表示された「OGP送信」「OGPコピー」の2つのブックマークレットをブックマークバーにドラッグして登録する（初回のみ）
3. 取得に失敗しているURLの一覧から「開く」をクリックし、対象ページを新しいタブで開く
4. 開いたページ上で「OGP送信」ブックマークレットをクリックする。メタ情報があなたのWikiに送信され、キャッシュが修復される
5. まれにフォーム送信をブロックするサイト（CSPが厳格なサイト）では「OGPコピー」でJSONをコピーし、管理ページの貼り付け欄から修復する

エラーになっていないURLのカードを更新したい場合も、管理ページの「任意のURLを修復・更新」から同じ手順で使えます。

**セキュリティ設計**: 管理ページはPukiWiki標準の `pkwk_login()`（`$adminpass`）で認証します。ブックマークレットからの送信は、ログイン時に発行される有効期限付きトークン（既定2時間、`PLUGIN_OGP_REPAIR_TOKEN_TTL` で変更可）で認証されます。トークンはサーバーにはSHA-256ハッシュのみ保存されます。また、画像はURLだけを受け取ってサーバー側で取得・画像形式を検証するため、任意ファイルを書き込まれる心配はありません。トークンの有効期限が切れたら、管理ページの「トークン再発行」を押してブックマークレットを登録し直してください。

## 変更履歴

### ver1.0（2019.9.10）[ ](https://oncologynote.jp/?12173bf026#j047861e)

ひとまずOGPを取得して表示する機能を実装しました。

### ver1.1（2019.9.17）[ ](https://oncologynote.jp/?12173bf026#z7aaf86d)

キャッシュ機能を実装しました。CACHE.DIRのogpというサブフォルダにキャッシュを配置します。またキャッシュ機能を実装したことでHTTPS外へのリンクを張った際に「安全でないサイト」と表示される問題も初回キャッシュ作成時以降は表示されなくなり、（ほぼ）解決しました。

しかし古いキャッシュを破棄する機能はまだ実装していません。また画像の拡張子を判断するステップを省略しているため、jpegもPNGも'.img'としています。主要ブラウザでは問題なく表示されるようですが、いずれ修正しなければならない。

また、URLを省略したOGPを提供するサイト（NCBI Pubmedなど）へのリンクを貼れるようにしました。

### ver1.2（2019.11.10）[ ](https://oncologynote.jp/?12173bf026#w45f8d9b)

第2引数にampを入れることでimgタグをamp-imgに変更できるようにしました（ver1.5から再びAMP非対応になっています）。

AMP対応したサイトの場合はimgタグを使用することができません。そのためOGPプラグインが書き出すタグもimgではなくamp-imgタグを使用するようにしています。これにともなってスタイルシートも若干複雑になります。ver1.2対応のスタイルシートを使用してください。

### ver1.3 (2020.5.2)[ ](https://oncologynote.jp/?12173bf026#sff2b6b0)

ファイル形式（GIF・PNGなど）を反映したキャッシュファイル拡張子になるようにしました。従来のキャッシュも利用できます。なお、ファイル形式を取得するために[getimagesize()](https://www.php.net/manual/ja/book.image.php)を使用しているため、PHPでGDがサポートされている必要があります。お使いのサーバーのPHPの設定をご確認ください。

### ver2.0 (2025.7.19)

- HTTP クライアントの強化: OpenGraph ライブラリで使う User Agent を Chrome 相当に更新し、Accept ヘッダーや gzip 展開、HTTP ステータスのチェックを追加しました。必要に応じて `PLUGIN_OGP_HTTP_USER_AGENT` で UA を差し替えられます。
- メタデータの補完: Twitter Card (`twitter:image`, `twitter:image:src`, `twitter:image:url`) や `<link rel="canonical">` も解析対象に加え、`og:image:secure_url` などの優先度を考慮してタイトルや画像の欠落を補うようにしました。
- 相対 URL・スキーム省略 URL 対応: `<base>` タグや `//example.com` のような表記でも確実に絶対 URL へ変換し、画像取得に失敗しにくくしました。
- WebP fallback の仕組みを刷新し、WebP と従来形式を並行キャッシュして `<picture>` で安全にフォールバックできるようにしました。古いキャッシュからでも自動で WebP 版を生成します。

### ver3.0 (2026.7.8)

- **取得失敗時のプレーンリンク・フォールバック**: OGP取得に失敗してもリンクが消えず、URLをそのまま表示するリンクカードになります。失敗はエラーキャッシュとして記録され、30日間は再取得を抑制します。
- **repairモード**: `?plugin=ogp` の管理ページとブックマークレットにより、取得に失敗したURLのキャッシュを自分のブラウザ経由で修復できるようになりました（詳細は上記「repairモード」参照）。
- **スキップドメイン**: `PLUGIN_OGP_SKIP_DOMAINS` で取得を試みないドメインを指定できます。
- 画像取得を `file_get_contents` から cURL（リダイレクト追従・タイムアウト・HTTPステータス検証付き）に変更しました。
- 画像でないデータを画像キャッシュとして保存しないよう検証を厳格化しました。
